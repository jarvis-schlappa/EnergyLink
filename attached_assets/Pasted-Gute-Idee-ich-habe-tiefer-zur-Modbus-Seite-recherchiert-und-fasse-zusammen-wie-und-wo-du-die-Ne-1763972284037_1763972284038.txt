Kurzüberblick — wichtigste Erkenntnisse
	•	E3/DC bietet eine Modbus/TCP-Schnittstelle (zwei Modi: E3/DC Simple-Mode und SunSpec-Mode). Modbus muss im Gerät aktiviert werden (Hauptmenü → Funktionen → Modbus / Modbus TCP).  ￼
	•	In der Praxis berichten Anwender, dass die aktuelle Phasen-Frequenz auf ihren Systemen im Registerbereich 41024 / 41025 liegt — typischerweise mit Skalierungsfaktor 0,01 (d.h. Registerwert 5000 → 50,00 Hz). Viele Threads sagen: wenn 41024 nichts liefert, probiere 41025 (bei einigen war das die Lösung).  ￼
	•	Einige Setups nutzen stattdessen SunSpec-Mapping; SunSpec-Modelle enthalten Frequenz-/Trip-Felder (für Schutz-/Droop-Einstellungen etc.). Je nach Modus/ Firmware kann die Frequenz also unter Simple-Mode an anderer Adresse und im SunSpec-Mode in SunSpec-Modellen liegen.  ￼
	•	In der Praxis schwanken Anwenderberichte: Firmware-/Release-Unterschiede führen dazu, dass Registeradressen sich ändern oder in unterschiedlichen Dokumenten/Foren anders genannt werden — teste deshalb beide Register (41024 + 41025) und kontrolliere Firmware/Modus.  ￼

Konkrete praktische Hinweise — wie du prüfen / auslesen solltest
	1.	Modbus aktivieren im S10/Hauskraftwerk: Menü → Funktionen → Modbus / Modbus TCP einschalten; Protokoll wählen (E3/DC Simple-Mode oder SunSpec). (Siehe offizielle Modbus-Kurzanleitung).  ￼
	2.	Test: beide Register lesen
	•	Viele Anwender nennen: 41024 oder 41025 = Phasen-Frequenz L1 (Unit: Hz, Skalierung ~0.01). Wenn eines kein Ergebnis liefert, das andere probieren. Forenbeitrag: Fehler gelöst, indem man statt 41024 das Register 41025 verwendet.  ￼
	3.	Adressierung / Tools
	•	Achtung Modbus-Adressnotation: Manche Tools akzeptieren „41025“ direkt; andere erwarten Null-basierte Adresse (z. B. 41025 − 40001 = 1024 als Startadresse). Passe Adresse an dein Tool an (modpoll, pymodbus, Node-RED-Modbus, ioBroker, etc.).  ￼
	4.	Skalierung/Umrechnung
	•	Wenn Registerwert z. B. 5000 zurückkommt, entspricht das bei Faktor 0.01 → 5000 * 0.01 = 50.00 Hz. Teste auf plausiblen Wertebereich (~45–55 Hz). (Skalierungsfaktor wird in Benutzerberichten und bei SunSpec/Modbus-Mappings erwähnt.)  ￼
	5.	SunSpec-Mode
	•	Falls du den SunSpec-Mode verwendest: SunSpec-Modelle enthalten Frequenz-/Trip-Felder (z.B. für Frequency Trip / Droop), die an anderen Offsets liegen. Wenn Simple-Mode nicht liefert, wechsle testweise in SunSpec-Mode und prüfe SunSpec-Register/Modelle.  ￼

Beispiel: konkreter Lesetest (2 Varianten)

A) Mit dem Kommandozeilen-Tool modpoll (häufig genutzt zum schnellen Test)
	•	Beispiel (wenn dein Gerät Modbus-ID 1 hat; Port 502):

modpoll -m tcp -r 41025 -c 1 -a 1 <IP_DEINER_E3DC> 502

(Je nach Implementation musst du evtl. statt -r 41025 die Adresse 1024 verwenden — also -r 1024 — because many tools use zero-based addressing.)
Wenn modpoll z. B. 5000 zurückliefert → reale Frequenz = 5000 * 0.01 = 50.00 Hz.

B) Kurzes Python-Beispiel mit pymodbus (lesend, Simple-Mode, Address-Offset beachten):

from pymodbus.client.sync import ModbusTcpClient

IP = "192.168.1.100"
PORT = 502
UNIT = 1  # Modbus device id

client = ModbusTcpClient(IP, port=PORT)
client.connect()

# Beispiel: wenn dein Tool 0-basierte Adresse erwartet:
addr = 41025 - 40001  # = 1024
rr = client.read_holding_registers(addr, 1, unit=UNIT)
if not rr.isError():
    raw = rr.registers[0]
    freq_hz = raw * 0.01   # Skalenfaktor 0.01 (prüfen!)
    print("Rohwert:", raw, "-> Frequenz:", freq_hz, "Hz")
else:
    print("Fehler beim Lesen:", rr)
client.close()

Hinweis: manche Geräte liefern 16-bit signed/unsigned oder kombinierte 32-bit Werte — prüfe Datentyp im konkreten Register und in deiner Firmware-Dokumentation.  ￼

Vorsicht / Troubleshooting (häufige Probleme)
	•	Kein Wert / 0 / kein Datensatz: Firmware-Version oder Modus (Simple vs SunSpec) kann Einfluss haben — update/prüfe Firmware-Stand und aktiven Modus. Teste beide Modi.  ￼
	•	Register verschoben: Community-Threads berichten, dass Adressen zwischen Releases gerückt wurden (deshalb Tipp: 41024 + 41025 testen).  ￼
	•	Skalierungsfaktor: normalerweise 0.01 in Nutzerberichten; aber prüfe Rückgabewerte auf Plausibilität. Bei SunSpec ist ein Skalierungs-Feld (Hz_SF) üblich.  ￼
